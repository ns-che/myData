HTML 기본 태그
<!DOCTYPE html>: 브라우저가 HTML을 어떻게 해석(렌더링)할지 알려줌(Standards Mode(최신 표준) vs Quirks Mode(옛날 과거 브라우저 호환용) vs Almost Standards Mode(Quirks Mode에 일부 레이아웃만 표준)) => HTML버전은 상관없이 모드는 3가지 존재
최신HTML을 위해서는 <!DOCTYPE html>활용
태그가 없을경우 Quirks Mode

<html lang="ko">: 검색엔진이나 이 정보를 보고 콘텐츠 관리(ex) 한국 사용자 우선 노출) 를 하고, 브라우저에서 이 정보를 보고 폰트, 맞춤법, 음성 지원(ex) 한국 발음 tts)을 함
<meta name="viewport" content="width=device-width, initial-scale=1.0">: 다양한 기기에서 화면 크기에 맞춰 웹페이지가 보이도록 설정 + 반응형 웹(css의 @media 쿼리 활용 가능해짐)
<meta name="keyword" content="keyword1, keyword2, 키워드3, kword4">: 웹 페이지 키워드 지정. 검색 엔진 최적화(SEO: Search Engine Optimization)에 사용 => 현재는 주요 외부검색엔진에서 사용되지 않음(검색 순위 조작으로 인해) => 내부 검색엔진, 문서정리에는 활용 가능
<meta name="description" content="">: 웹 페이지 간단한 설명. 검색 결과(SERP: Search Engine Result Page) 요약문(스니펫) 표시에 활용
<link rel="icon" type="image/svg+xml" href="favicon.svg" />: 브라우저에게 대표 아이콘 알려주는 데 활용됨
파비콘(favicon) = favorite icon = 웹사이트를 대표하는 작은 아이콘
확장자    MIME(Multipurpose Internet Mail Extensions) 타입
PNG	image/png
JPEG	image/jpeg
ICO	image/x-icon
SVG	image/svg+xml => svg(Scalable Vector Graphics)는 XML(Extensible Markup Language) 기반의 이미지 형식
웹 브라우저는 확장자가 아니라 MIME 타입으로 파일 종류를 인식함

<link> => 리소스를 연결하는 태그. 브라우저가 파일의 성격을 알 수 없음(이미지인지 스타일시트인지)
=> link는 브라우저가 리소스를 활용할 수 있도록 이미 정해진 이름에 매핑시키는 역할이다.

10) App.js
function App(){return()} vs const App = ()=>{return()} 일반함수(Function Declaration) vs 화살표 함수(Arrow Function)의 차이
=> 호이스팅(Hoisting: 국기게양할때 그 게양)과 this가 참조하는 위치가 다름
호이스팅: 함수 선언전 호출 가능한지 여부 => 화살표 함수는 변수에 할당하는 형태라서 함수가 존재하는지 인식할 수 없음
this 참조 위치: 화살표 함수는 this를 만들지 않고 호출된 위치에서의 this를 활용

변수(재사용 => 일관성, 수정용이), 중첩(Nesting 구조화 => 가독성), Mixins(코드 블록 재사용), 확장(Extend: 스타일 상속 => 중복방지와 재사용), 연산(수학 연산 지원 => 일일이 사람이 계산을 먼저 한 이후 작성하지 않아도 됨)

3) style.scss
@charset "UTF-8";  문자 인코딩. 해당 SCSS파일이 UTF-8임을 알림
@import 'setting/fonts'; 폰트
@import 'setting/vars'; 변수
@import 'setting/reset'; 브라우저 스타일 초기화/리셋
@import 'setting/mixin'; 믹스인(스타일 재사용)
@import 'setting/common'; 공통 css

=> _이름.scss 형태로 작성할 경우 partial 파일임 => 빌드시 따로 CSS 파일로 변환되지않고 내용이 import된 곳에 병합됨
