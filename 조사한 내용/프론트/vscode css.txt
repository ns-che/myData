2. 값 단위
1) 크기 단위: width, height, max-width, max-height
- 절대 단위
px(픽셀), cm, in

- 상대 단위
%: 부모태그 기준 백분율 => 브라우저 크기에 따라 변동되지 않음
vw(viewport width): 웹 브라우저 너비 기준 백분율 설정 => 브라우저 크기에 따라 변동
vh(viewport height): 웹 브라우저 높이 기준 백분율 설정 => 브라우저 크기에 따라 변동
vmin: 웹 브라우저 너비 높이 중 작은 것 기준 백분율 설정 => 브라우저 크기에 따라 변동
vmax: 웹 브라우저 너비 높이 중 큰 것 기준 백분율 설정 => 브라우저 크기에 따라 변동
em

2) 위치 단위: position(top, left, right, bottom), padding, margin
px, em, %
auto: 남는 공간을 브라우저가 자동 계산 => 브라우저 크기에 따라 변동
=> 보통 margin 0 auto; 형태로 많이 씀 => 상하 여백 없음, 좌우 여백은 (브라우저 크기-블록 크기)/2 양옆 분배 => 수평 가운데 정렬

참조: https://aram5.tistory.com/9

3. 포지션
static	기본값, normal flow, top/left/right/bottom 이용불가
relative	자기 자리 기준 이동, normal flow
absolute	부모 기준 절대 위치, normal flow 아님
fixed	뷰포트 기준 절대 위치, normal flow 아님 => 스크롤 내리거나 해도 딱 붙어있음

부모에 position을 명시적으로(static은 제외) 주는 이유는, 자식 absolute 요소의 기준을 명확히 부모로 지정하기 위해서
=> 지정하지 않을 경우 viewport(root)가 부모가 되버림
=> 그 중 relative는 static과 마찬가지로 normal flow를 그대로 가져가므로 ui에 변화가 없음 => 많이 사용되는 이유라 생각

normal flow : 기본 규칙대로 순서에 맞춰 배치
block: 수직으로 쌓임(크기(width, height) 지정 가능)
inline: 수평으로 이어짐(크기 지정 불가) => margin padding은 글자 블록(line-box) 기준(line-height - https://velog.io/@ursr0706/line-height-%EC%86%8D%EC%84%B1
inline-block : 수평으로 이어짐(크기 지정 가능)

디폴트 값
div	block
p	block
span	inline
img	inline-block
section	block
button	inline-block

부모가 인라인(display: inline)이라도 자식이 block이면 블록의 특성을 가짐
